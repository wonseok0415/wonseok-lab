## 3. 파일 명명 규칙 (Naming Convention)

**기본 형식**: `스테이빌리티_양평_<문서종류>_<버전>_<날짜>.<확장자>`

| 문서 종류 | 파일명 예시 |
|---|---|
| 도면 (PNG) | `스테이빌리티_양평_IoT배치도_v8.png` |
| 논의 정리 (XLSX) | `스테이빌리티_양평_AI홈솔루션_정리_v2_20260527.xlsx` |
| 시나리오 (XLSX) | `스테이빌리티_양평_시나리오_v2_20260527.xlsx` |
| 종합 정리 (MD) | `스테이빌리티_양평_논의정리_v2_20260527.md` |
| 종합 정리 (HTML) | `스테이빌리티_양평_논의정리_v2_20260527.html` |

**규칙**:
- 날짜는 `YYYYMMDD` (예: `20260527`)
- 버전은 `v숫자` (도면은 v1~v8까지 진행됨, 다음은 v9)
- 한국어 파일명 사용 (영문 변환 금지)
- 공백 대신 언더스코어(`_`) 사용

## 5. 작업 원칙 (Working Principles)

> ⚠️ **본 섹션은 사용자가 반복적으로 강조한 규칙들이며, 새 세션에서도 반드시 준수해야 합니다.**

### 5.1 데이터 출처 엄격 준수
- **"어나더빌라 양평" 탭의 정보만** 양평 프로젝트 자료로 사용
- 홍천(1호점) 등 다른 탭의 정보를 **양평에 가져와 추정하거나 보완 금지**
- 자료가 없으면 "미확정" 또는 ⚠️로 표기, 임의로 채우지 않음

### 5.2 가정 명시 / 보수적 접근
- 도면·자료에 명시되지 않은 사항은 **"가정" 또는 "미확정"으로 명확히 표기**
- 예: 도면의 공간명과 출품목 탭 공간명 매칭 시, 사용자 컨펌 전까지는 "(마스터 베드룸1?)" 등 물음표 표기
- 사용자 컨펌을 받은 후에만 가정 표기 제거

### 5.3 미확정 항목 명시 (⚠️)
- 모델 미확정, 위치 미정, 적용 여부 미정 → 모두 ⚠️ 표기
- 엑셀: 노란색 행 `#FFF2CC` 하이라이트
- HTML: `.warning-row` 클래스 사용

### 5.4 도면 충실 반영
- 실제 도면의 룸 비율을 시각적으로 정확히 반영 (v3 → v8까지 점진 보정)
- 도면에 표기된 A/E 마커는 원본 그대로 유지
- 임의로 공간(예: 야외 수영장, BBQ 데크 등) 추가 금지

### 5.5 SKS 자동화 제한 명시 의무
- **SKS (Signature Kitchen Suite / LG SIGNATURE 2세대) 라인업은 ThinQ 연동/스마트루틴 제한 가능성** 명시
- 모든 산출물 상단 또는 주요 위치에 빨간 배너로 안내
- 적용 범위는 HS사업부 확답 전까지는 "가능성"으로만 표기

### 5.6 사용자 정정 사항 반영 (반드시 유지)
- **G1 = G8**: 써모스탯이 보일러 제어를 통합 수행 → 별도 보일러 연동 모듈 불필요
- **G6, G7는 3rd 파티 제품**: 전동 샷시(LX하우시스), 전동 커튼(마마바-Matter)
- **양평 가전 모델명**: 추후 업데이트 예정 → 카테고리/제품군만 표기
- **팬트리**: 도면 안에 위치 / 상세 위치는 미정
- **관리실**: 외부에 위치 → 본 검토 범위 외
- **공간명 매핑 컨펌됨**: 침실-1·2 ↔ 마스터 베드룸1·2, 화장실-1·2 ↔ 외욕실·내욕실
- **외/내욕실 바스에어**: 양평 탭 모델 오기재 → LG 퓨리케어 바스에어시스템 프리미엄 듀얼 (MX0120BASV)

### 5.7 산출물 자기완결성
- Excel/HTML은 외부 의존성 없이 독립 실행 가능해야 함
- HTML CSS는 인라인 임베드 (외부 CDN 사용 금지)
- 한국어 폰트 처리: matplotlib는 `/usr/share/fonts/Batang.ttf` + `FontProperties(fname=...)` 사용
